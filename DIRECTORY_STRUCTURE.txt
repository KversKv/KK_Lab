================================================================================
                        KK_Lab 工程目录结构说明
================================================================================

KK_Lab 是一个基于 PySide6 (Qt) 的芯片功耗测试与调试桌面工具，通过 VISA/串口
协议控制多种实验室仪器（电源分析仪、示波器、温箱等），实现自动化数据采集、
PMU 测试、Charger 测试、功耗分析和波形展示等功能。

================================================================================
  顶层目录
================================================================================

KK_Lab/
│
├── main.py                  # 应用程序主入口，初始化 QApplication 并启动主窗口
├── log_config.py            # 日志配置模块，提供 setup_logging() 和 get_logger()
├── debug_config.py          # 调试配置，DEBUG_MOCK 开关控制是否使用模拟仪器
├── requirements.txt         # Python 依赖包清单 (PySide6, pyqtgraph, pyvisa 等)
├── .gitignore               # Git 忽略规则
├── DIRECTORY_STRUCTURE.txt  # 本文件 - 工程目录结构说明
│
├── core/                    # 核心业务逻辑层
├── instruments/             # 仪器驱动层 (按仪器类型分包)
├── ui/                      # 用户界面层 (按功能分包: widgets/pages/dialogs/styles)
├── lib/                     # 第三方库与底层接口封装
├── resources/               # 静态资源文件
├── helps/                   # HTML 帮助文档 (各功能页面的操作说明)
├── hooks/                   # PyInstaller 自定义 hook
├── spec/                    # PyInstaller 打包配置
└── Results/                 # 测试结果输出目录 (运行时生成，Git 不跟踪)

================================================================================
  core/ - 核心业务逻辑
================================================================================

core/
├── test_manager.py          # 测试管理器，协调数据采集流程，管理测试生命周期，
│                            # 发出数据更新信号供 UI 层订阅
├── data_collector.py        # 数据采集器，基于 QTimer 定时从仪器读取电流/电压
│                            # 数据，支持可配置采样率
└── controllers/             # 控制器子包
    └── oscilloscope_controller.py  # 示波器控制器，封装连接/断开/测量/截图/
                                    # 设置应用等高层操作
