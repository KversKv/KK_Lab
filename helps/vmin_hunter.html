VminHunter（最低工作电压探底）

页面介绍

VminHunter 页面用于探测芯片能稳定工作的最低电压边界（Vmin）。系统按照电压点列表从高到低逐点施加电压，结合 UART LOG 心跳与关键字进行死机检测，从而确定芯片可稳定运行的最低电压。

测试模式

  • Internal Voltage (IIC) — N6705C 作为电压表测量真实电压，通过 IIC 控制字调节芯片内部电压，需要电压校准。
  • External Supply (N6705C) — N6705C 直接作为电压源供电，无需校准。

配置说明

  • Test CNT — 每个条件下的测试次数（循环次数）。
  • Temperature Control — 启用后可填写温度点列表（°C），配合温箱遍历多温度条件。
  • Monitor Channels — VcoreM 为必选监测通道；VcoreL 为可选监测通道。
  • Voltage Points — 需要遍历的电压点（V），建议由高到低排列。
  • Channel Config — 配置 N6705C 监测通道及 IIC 控制字（设备地址、位宽、睡眠/唤醒电压控制地址）。

测试方法

  1. 连接 N6705C 与 UART（DUT Log）。
  2. 选择测试模式并配置电压点、监测通道与 IIC 控制字。
  3. （可选）启用温度控制并填写温度点。
  4. 点击 Start Hunt 开始遍历，系统按电压点逐点测试并实时刷新结果表。
  5. 当某电压点检测到死机（心跳超时或崩溃关键字）时，记录失败并触发 IO 恢复（上电/复位）。
  6. 遍历结束后，可在结果区查看各电压点状态与最终 Vmin，并导出结果。

死机检测与恢复

  • 心跳超时 — 在设定时间内未收到 UART 心跳判定为死机。
  • 崩溃关键字 — UART LOG 中出现崩溃关键字判定为死机。
  • IO 恢复 — 检测到死机后通过 IO 控制对芯片进行上电或复位以恢复测试。

注意：本页面当前为完整 UI 与配置交互层；遍历引擎、电压校准与死机判定/恢复的核心编排将在 core 层通过 QThread 异步实现。
