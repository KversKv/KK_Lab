# PMU 测试项优缺点与完整度分析

> 范围：基于 `docs/user/pmu_test_reports/` 下 7 份历史 PMU 测试报告。
> 配套测试项矩阵见 [00_PMU测试用例与方法汇总.md](./00_PMU测试用例与方法汇总.md)。
> 目的：评估现有 PMU 测试项体系的优点、缺点与完整度，给出改进方向（面向带 AI 助手的自动化测试工具）。

---

## 1. 总体评价

现有 PMU 测试报告**覆盖面较全、工程经验密集**，已形成一套相对稳定的"PMU 常规测试"骨架
（开关机/复位 → 电源质量 → 时钟/ADC/接口 → GPIO/保护）。但报告以**人工抓波形 + 飞书外链**为主，
**结构化、判定标准、可复现性、自动化程度**偏弱，难以直接被工具/AI 复用。

完整度评分（主观，满分 5）：

| 维度 | 评分 | 说明 |
|---|:--:|---|
| 测试项覆盖广度 | 4.5 | 关键项基本齐全，1811/2800 最全 |
| 测试方法描述 | 3.0 | 多数有方法但条件零散，部分仅放波形图 |
| 判定标准明确度 | 2.5 | 仅 GPADC、GPIO 脉冲等少数项有量化判定 |
| 数据结构化 | 2.0 | 大量数值散落正文/外链，难聚合对比 |
| 可复现性 | 2.5 | bin 路径/寄存器零散，缺统一前置条件清单 |
| 自动化潜力 | 3.0 | 电压扫描/效率/瞬态适合自动化，已有页面支撑 |

---

## 2. 优点

1. **测试项体系成熟**：开关机时序、瞬态电流、纹波、PSRR、效率、GPADC、挡位扫描、保护电路等
   形成稳定清单，跨 7 颗芯片高度一致，可直接抽象为通用矩阵（见汇总 §2）。
2. **工程经验沉淀丰富**：记录了大量"坑点 + 处理办法"，例如：
   - SIMO（1307PH）输出切换导致纹波偏大，去掉过冲后平均纹波可减半；
   - 1503P A 版 LDO_VCORE_L 过冲/掉坑，B 版 0.6V 方案解决；
   - 2720 大小 BG 切换影响同 Vbit 输出，需调低 lp_bias；
   - 1806P BUCK ramp 切换过冲稳定复现、掉坑概率复现。
3. **跨芯片差异点清晰**：明确标注各芯片特殊约束（2008H ≤3.3V、08/08H 不支持 PATTERN 复位、
   1811 含 Charger/Fuel Gauge/DVFS、合封未出单线通信口等）。
4. **关键参数有实测值**：耐压、阻抗、死区时间、LPO 频率范围、复位时序等均有具体数字，便于建库。
5. **含可靠性/边界项**：Latch Up、LX 负压、开关机撞沿、VMIC 底噪等非常规但重要的项已覆盖。

---

## 3. 缺点 / 不足

1. **判定标准缺失**：除 GPADC（±10mV）、GPIO 脉冲（<50mV 无毛刺）外，多数项无明确 pass/fail 阈值，
   "OK"靠人工目视波形判断，无法自动判定。
2. **数据非结构化**：纹波、效率、调整率等关键数值大量以波形图/飞书外链承载，正文缺表格，
   无法横向聚合与回归对比；外链存在失效风险。
3. **前置条件零散**：bin 文件路径、寄存器地址、负载/限流设置、Vref 等散落在正文，
   缺统一"前置条件 + 仪器配置"清单，复现成本高。
4. **覆盖不均衡**：1811/2800 接近全覆盖，而 1503P/2008H 多项留空（PSRR、挡位、瞬态等"待测/空白"），
   各报告完成度参差，难判断是"未测"还是"通过未记录"。
5. **缺统一模板与版本信息**：各报告字段、章节顺序、命名不一致；缺测试日期/版本/测试人/环境元数据
   （仅个别如 2720 标了测试日期与 bin）。
6. **高低温 / 长期老化覆盖弱**：高低温仅部分项（GPADC、32K、LPO）涉及，缺系统化温度矩阵。
7. **自动化衔接缺位**：报告未与本工具页面（pmu_output_voltage / pmu_dcdc_efficiency / pmu_gpadc /
   pmu_oscp / pmu_clk / pmu_is_gain）的可执行测试项建立映射。

## 4. 完整度差距清单（建议补齐）

| 类别 | 现状 | 建议补齐 |
|---|---|---|
| 判定标准 | 多数项无阈值 | 为每个测试项定义量化 pass/fail（纹波 mVpp、效率 %、调整率 %/V、过冲 mV 等） |
| 数据记录 | 散落正文/外链 | 统一结构化表格（挡位→电压→步进→线性度；负载→效率等），数据随报告留存 |
| 前置条件 | 零散 | 每项补"前置条件 + 仪器通道 + bin/寄存器 + 负载/限流"清单 |
| 元数据 | 缺失 | 补测试日期 / 芯片版本 / 测试人 / 温度 / 环境 |
| 温度矩阵 | 部分项 | 关键电源项补常温/高温/低温三态 |
| 可靠性 | 部分芯片 | Latch Up / LX 负压 / 撞沿统一纳入每颗芯片必测 |
| 工具映射 | 无 | 把可自动化项映射到对应 PMU 页面与快捷指令 |

---

## 5. 面向 AI 助手 / 自动化工具的改进建议

1. **测试项标准化**：以本目录 [00_PMU测试用例与方法汇总.md](./00_PMU测试用例与方法汇总.md) 的矩阵为基线，
   形成"通用测试项 + 量化判定 + 前置条件"模板，新芯片直接套用，避免重复造章节。
2. **沉淀进 AI 记忆体系**：通用测试项 / 跨芯片差异 / 共性经验已写入伞目录
   `docs/kk_lab_ai_memory/automation/pmu_test/`（总记忆），页面专属项分散写入各 `pmu_*` 页面目录（分记忆）。
3. **数据结构化先行**：把电压挡位扫描、DCDC 效率、瞬态/调整率等高频项的结果以表格沉淀，
   便于 AI 做回归对比与异常识别。
4. **判定自动化**：对有明确阈值的项（GPADC ±10mV、GPIO <50mV、纹波/效率阈值）实现自动判定，
   减少人工目视。
5. **前置条件参数化**：bin、寄存器、负载、限流、Vref 等作为参数模板，供 AI 助手在对应页面一键填充执行。
