# 滑动窗口事件检测算法（SWED：Sliding-Window Event Detection）

> 面向功耗/电流波形的轻量级事件分段算法。核心：固定时长滑动窗口 + 增量均值 + 单调队列极值 + 双判据触发。
> 适用：海量原始点（数十万~数千万）单趟流式扫描，O(N) 复杂度，恒定内存，零第三方依赖。
> 关联实现位：`core/ai/providers/waveform_provider.py`（与 STA-LTA / PELT 双引擎互补，详见 `core/ai/WAVEFORM_ANALYSIS_SESSION.md` §9）。

---

## 0. 一句话定义

以固定时长 `T` 的窗口沿时间轴滑动，**增量**维护窗口均值与窗口极值；当**均值发生相对突变**或**极值显著偏离均值**时判定为事件起点，再向后定界得到事件段，输出「起始时间 + 段平均电流」。

---

## 1. 符号与输入

| 符号 | 含义 |
|---|---|
| `x[0..N-1]` | 原始采样值序列（电流，单位 A 或内存态 mA） |
| `t[0..N-1]` | 对应时间戳（s），可等步长或非等步长 |
| `dt` | 采样间隔 = `median(diff(t))`，采样率 `fs = 1/dt` |
| `T` | 窗口时长（s），用户设定 |
| `W` | 窗口点数 = `max(1, round(T / dt))` |
| `n` | 窗口序号（窗口左端点索引），`n = 0 .. N-W` |
| `AVG_Tn` | 第 `n` 个窗口 `x[n .. n+W-1]` 的算术平均 |
| `Min_Tn` / `Max_Tn` | 第 `n` 个窗口内的最小值 / 最大值 |

> 注：原始描述中「AVG_Tn 代表当前 T 第一个点对应的 AVG」——规范化为：`AVG_Tn` = 以索引 `n` 为**左端点**、长度 `W` 的窗口均值。窗口每滑动 1 点 `n→n+1`。

---

## 2. 三个动态量的增量维护（核心性能来源）

### 2.1 滑动均值 AVG —— 增量 O(1)/步

维护窗口内元素和 `S_n = Σ x[n..n+W-1]`：

```
S_0   = x[0] + x[1] + ... + x[W-1]          # 初始化，一次 O(W)
S_n   = S_{n-1} - x[n-1] + x[n+W-1]         # 滑动一步，O(1)：减移出点、加新进点
AVG_Tn = S_n / W
```

- 与原始构想一致：**不重算整窗，只做一减一加**。
- ⚠️ **浮点累积误差**：长序列下 `S_n` 反复加减会累积舍入误差。工程对策——每滑动 `R`（如 `R = 2^16`）步用窗口内真实求和**重算校正**一次；或用 Kahan/Neumaier 补偿求和。代价可忽略（每 65536 步一次 O(W)）。

### 2.2 滑动极值 Min/Max —— 单调双端队列，摊还 O(1)/步

> ⚠️ **关键修正**：极值**不能**用「减移出点」做增量。若移出的点恰是当前 `Min`，无法 O(1) 得知新最小值。正确做法是经典 **Sliding Window Minimum/Maximum**，用**单调双端队列（monotonic deque）**：队列存索引，值单调，队首即当前极值。

**最大值队列 `Dmax`（值单调递减）**：

```
入队 i：当 Dmax 非空且 x[Dmax.back] <= x[i]，弹出队尾   # 维持递减
        Dmax.push_back(i)
过期  ：当 Dmax.front < n（已滑出左边界），弹出队首
取值  ：Max_Tn = x[Dmax.front]
```

**最小值队列 `Dmin`（值单调递增）** 对称处理，`Min_Tn = x[Dmin.front]`。

- 每个点最多入队 1 次、出队 1 次 → 全程 `2N` 次操作 → **摊还 O(1)/步、整体 O(N)**。
- 内存：两条队列最坏 `O(W)`，恒定上界。

> 这是工业界标准做法（LeetCode 239 / 滑动窗口极值），数学上保证正确，且与「不重算整窗」的初衷一致——只是载体从「标量和」换成「单调队列」。

## 3. 事件判据（双触发）

设当前窗口均值 `AVG_Tn`，上一窗口均值 `AVG_T(n-1)`，并维护一个**事件基线** `BASE`（见 §3.3）。

### 3.1 判据 A：均值相对突变（电平/平台切换）

```
若 |AVG_Tn - AVG_T(n-1)| / max(|AVG_T(n-1)|, ε) >= 0.20    → 触发「均值突变事件」
```

- `ε`（小正数，防除零，如 `1e-9` 或基线噪声地板）保护静默段 `AVG≈0` 时的爆炸。
- 抓的是**台阶/平台/状态切换**（RX、DVFS、负载变化）。
- 阈值 `theta_avg = 0.20`（20%）可配。

### 3.2 判据 B：极值显著偏离均值（窄尖峰/突发）

```
若 (Max_Tn - AVG_Tn) / max(|AVG_Tn|, ε) >= 0.35           → 触发「上冲尖峰事件」
或 (AVG_Tn - Min_Tn) / max(|AVG_Tn|, ε) >= 0.35           → 触发「下陷事件」
```

- 抓的是**涌流/浪涌等窄脉冲**，即使均值还没明显变，极值已暴露异常。
- 阈值 `theta_ext = 0.35`（35%）可配。
- ⚠️ **静默段误触发**：睡眠基线 µA 级抖动时，任何微小纹波相对值都可能 >35%。**必须叠加睡眠门限闸门**（`sleep_floor`，见 §3.4），低于门限即睡眠态、免疫所有判据，否则睡眠段刷屏伪事件——这是 WAVEFORM_ANALYSIS_SESSION §9.7 已验证的坑。

### 3.3 事件基线 BASE 与防抖（避免一个事件反复触发）

朴素地"逐窗比较 AVG_Tn 与 AVG_T(n-1)"会让**一个持续事件每步都触发**。规范化为「状态机 + 基线锁定」。

> **设计演进：双线驱动（当前实现）。** 早期为单状态机 + 耦合结束：
> `结束 = (相对回落 或 睡眠) 且 极值平静`。该耦合的副作用——在 RX 台阶上，第一个尖峰开启事件后，
> 均值停在台阶水平（既不回落也不睡眠），事件永不结束，台阶内后续尖峰簇全被吞进同一个事件
> （宽窗 `T` 下尤甚，4 个簇糊成 1 个）。
>
> 改为**双线平权、各自独立开关事件**（均值线 + 极值线，互不绑架）：
>
> **均值线（level）**：
> ```
> IDLE：命中判据 A（hitA）→ 进入 IN_EVENT，type=level，记录 start
> IN_EVENT：维护 ev_max；ev_ref = max(ev_max, BASE, floor)
>           结束 = 相对回落 (AVG <= ev_ref*(1-theta_end)) 或 睡眠 (AVG < sleep_floor)
>           结束 → 更新 BASE，回 IDLE，冷却 W 点
> ```
> **极值线（spike/dip）**：
> ```
> IDLE：命中判据 B（hitB_up→spike / hitB_dn→dip）→ 进入 IN_EVENT，记录 start
> IN_EVENT：结束 = 极值平静 (up<theta_ext and dn<theta_ext) 连续持续 calm_hold(=W//2) 窗
>           或 睡眠
>           结束 → 回 IDLE，冷却 W 点
> ```
> 两线互不影响：RX 台阶被 level 线完整覆盖（均值不回落不被打断）的同时，台阶内尖峰簇被
> spike 线逐个独立切出。两线事件最终按 `start` 排序合并进**同一** events 列表，靠 `type`
> 字段区分；下游 `to_index_ranges()` 等消费零改动。`merge_gap` 仅合并相邻同型 spike/dip。

- **结束判据用「相对回落」而非「绝对 floor」**：早期版本用 `AVG < floor` 结束，但当 floor
  降为纯物理睡眠线（400 µA）后，持续活跃平台（如 test.csv 1.1~5 s 稳定 1.5 mA）始终高于
  floor，事件永不结束、被合成巨型事件。改为「均值回落到事件活跃水平的 `(1-theta_end)` 以下」
  即结束（`theta_end` 默认 0.50，即回落超 50%），睡眠兜底仅作为额外退出口。
- `BASE` 用「上一稳定段均值」而非「上一窗均值」，避免缓慢漂移被持续误判。
- `cooldown` 防止边界处来回跳变产生碎片事件。
- **持续稳定平台不被切碎是预期行为**：若一段电流长期平稳（既不回落、相对自身也无突变），
  它本就是一个连续 `level` 事件；平台内部的精细细分交由 PELT drill-down（见 §7.4）。

### 3.4 睡眠门限闸门 `sleep_floor`（单一物理变量）

```
floor = sleep_floor          # 默认 0.4 mA（= 400 µA），单一物理门限，无统计估计
asleep = (AVG_Tn < sleep_floor)        # 窗口均值低于门限即睡眠态
睡眠态下 hitA = hitB = False           # 完全免疫所有判据（含极值毛刺）
判据 A / B 仅当窗口越过 floor 才有效（滤掉睡眠基线微扰伪事件）
```

#### 3.4.1 为什么用单一物理门限，而不用 MAD 统计闸门

> **设计目标**：对 MCU/SoC，电流 < ~400 µA 通常是睡眠/关机态，此区测试抖动相对值极大
> （µA 级噪声 ÷ µA 级基线 → 相对波动轻易爆 20%/35%），是判据 A/B 误触发的根因。
> 需求很明确：**低于物理睡眠线的波动一律视为睡眠抖动、直接忽略**。
>
> **MAD 闸门为何被弃用**：MAD（`med + k·mad·1.4826`）是**全局统计量**，会被活跃段污染。
> 实测 test.csv（睡眠段 < 0.33 mA，但 0.4~1.1 s 有 2~33 mA 活跃段），全局 MAD floor
> 被抬到 **2.17 mA**——这条"自动估计的噪声线"远高于物理睡眠线，含义已失真，且不可由用户直观控制。
>
> **结论**：放弃 MAD，`floor` 直接等于 `sleep_floor` 这一个用户可直接理解、直接调节的物理变量：
>
> | 电流水平 | 判定 | 处理 |
> |---|---|---|
> | < 400 µA | 睡眠态（µA 级抖动） | 免疫所有判据，不产生事件 |
> | ≥ 400 µA | 活跃 | 正常参与判据 A / B |
>
> **实测（test.csv，mA 单位）**：`floor = 0.4 mA`，0~0.4 s（max 324 µA）零事件；
> 4 簇浪涌 + 2 涌流全中、峰值 33.27 mA 正确。
>
> **单位约定**：算法内部数值的单位由输入数据决定。本项目 test.csv 一列即 **mA**
> （如 `33.27` = 33.27 mA = 0.033 A，整体 < 1 A），故 `sleep_floor` 默认取 `0.4`（= 0.4 mA = 400 µA）。

## 4. 事件定界（找长度）

触发只给出**起点**，需向后确定**终点**。两类事件用不同定界策略：

### 4.1 均值突变事件（平台型）

```
进入 IN_EVENT 后，记录新平台目标均值 AVG_target（触发后若干窗稳定值）
逐窗前探，当 AVG_Tn 回落到 |AVG_Tn - BASE| / BASE < theta_avg/2（回归基线带，迟滞）
    → 事件结束，end = 当前 t[n]
迟滞（hysteresis）：进入阈 20%、退出阈 10%，避免边界抖动反复开合
```

### 4.2 极值尖峰事件（脉冲型）

```
起点 = 首次穿越 floor（或满足判据 B）的点
向前/向后扩展，直到连续 m 个点回落到 floor 以下 → 终点
峰值 = 区间内 max，宽度 = end - start
邻近合并：两尖峰间隔 < merge_gap（如 0.2ms）视为同一簇，合并上报
```

## 6. 完整流程（伪代码）

```text
W   = round(T / dt)
S   = sum(x[0..W-1]);  Dmax, Dmin 初始化(x[0..W-1])
AVG_prev = S / W;  BASE = AVG_prev
floor = sleep_floor                            # 单一物理门限（默认 0.4 mA），无需预扫
state = IDLE;  cooldown = 0

for n in 0 .. N-W:
    # ---- 增量更新（O(1) 摊还）----
    if n > 0:
        S += x[n+W-1] - x[n-1]
        deque_push(Dmax, Dmin, n+W-1); deque_expire(Dmax, Dmin, n)
    AVG = S / W
    Mx  = x[Dmax.front];  Mn = x[Dmin.front]

    # ---- 周期性误差校正 ----
    if n % R == 0:  S = sum(x[n..n+W-1])

    # ---- 判据（睡眠态免疫）----
    asleep = AVG < sleep_floor                       # floor=sleep_floor
    hitA = not asleep and abs(AVG - BASE)/max(abs(BASE),floor,eps) >= theta_avg
    hitB = not asleep and (Mx - AVG)/max(abs(AVG),floor,eps) >= theta_ext and Mx >= floor

    if state == IDLE and cooldown == 0 and (hitA or hitB):
        open_event(start=t[n], type= 'spike' if hitB else 'level')
        ev_max = max(AVG, BASE);  state = IN_EVENT
    elif state == IN_EVENT:
        ev_max = max(ev_max, AVG)
        ev_ref = max(ev_max, BASE, floor)
        fell_back = AVG <= ev_ref * (1 - theta_end)  # 相对回落（§3.3）
        if (fell_back or AVG < sleep_floor) and ext_calm:
            close_event(end=t[n], avg=segment_mean)
            BASE = AVG;  state = IDLE;  cooldown = W
    AVG_prev = AVG
    if cooldown > 0: cooldown -= 1
```

## 7. 复杂度与开销评估

### 7.1 时间复杂度

| 环节 | 单步 | 全程 |
|---|---|---|
| 滑动均值 `S` | O(1) | O(N) |
| 单调队列 Min/Max | 摊还 O(1) | O(N)（每点至多 1 入 1 出） |
| 判据 + 状态机 | O(1) | O(N) |
| 初始化 / 周期校正 | — | O(W) + O(N/R·W) ≈ O(N) |
| **总计** | **O(1) 摊还** | **O(N) 单趟** |

> 对比：朴素「每窗重算 min/max」是 O(N·W)，窗口越大越爆炸；单调队列把 W 因子彻底消掉，这是本算法相对朴素实现的**核心性能优势**。

### 7.2 空间复杂度

| 项 | 占用 |
|---|---|
| 滑动均值 | O(1)（一个累加器） |
| 两条单调队列 | 最坏 O(W)，典型远小于 W |
| 事件列表 | O(事件数)，远小于 N |
| **原始数据** | O(N)，可走 `memmap`/分块流式，不必全驻留 |

→ **恒定附加内存**，与 N 无关（除原始数据本身），天然适配 §8 的 GB 级流式场景。

### 7.3 实测量级估算（Python+numpy 向量化 / 纯 Python 对比）

| 数据规模 | numpy 向量化（滑窗和用前缀和 + 单调队列） | 纯 Python 标量循环 |
|---|---|---|
| 24 万点（5MB） | ~10–30 ms | ~150–300 ms |
| 700 万点（150MB） | ~0.3–0.8 s | ~5–10 s（不推荐） |
| 5000 万点（1GB） | ~2–5 s（分块流式） | 不可用 |

> 建议实现：均值用前缀和向量化；单调队列因含分支难纯向量化，可用 numba/Cython 或接受其 O(N) 常数。事件数稀疏，状态机分支命中率低，实际很快。

### 7.4 与现有双引擎的关系

| 维度 | SWED（本算法） | STA-LTA + MAD | PELT |
|---|---|---|---|
| 抓什么 | 均值突变(A) + 极值偏离(B) 通吃 | 窄尖峰/突发 | 均值台阶/平台 |
| 复杂度 | O(N) 单趟 | O(N) 单趟 | 近 O(N)，最坏 O(N²) |
| 参数 | T / theta_avg / theta_ext / k | sta/lta/on/off/k | pen / min_size |
| 定位 | 轻量「一遍扫」首选，全程总览 | 浪涌定位 | drill-down 平台细分 |

> SWED 的判据 A≈PELT 的均值变点直觉、判据 B≈STA-LTA 的突发直觉，**用一条 O(N) 扫描同时近似两者**，是「廉价的全局首扫」；高精度细分仍交给 PELT。

## 8. 已知风险与工程对策（务必落地）

| 风险 | 现象 | 对策 |
|---|---|---|
| 极值增量误解 | 「减移出点」对 min/max 不成立 | **必须用单调队列**（§2.2） |
| 浮点累积误差 | 长序列 AVG 漂移 | 周期重算校正 / Kahan 求和（§2.1） |
| 静默段误触发 | 睡眠基线 35% 判据刷屏 | 睡眠门限 `sleep_floor` + 睡眠态免疫（§3.4） |
| µA 级睡眠重尾抖动 | 毛刺穿透成大量伪事件 | 绝对睡眠门限 `sleep_floor`（默认 400 µA）+ 睡眠态免疫（§3.4.1） |
| 活跃段事件不收尾 | 绝对 floor 太低，持续平台始终高于 floor、合成巨型事件 | 结束判据改「相对回落」`theta_end`（§3.3） |
| 除零爆炸 | `AVG≈0` 时相对判据发散 | `max(|·|, ε)` 保护 |
| 单事件反复触发 | 持续事件每步都报 | 状态机 + BASE 锁定 + cooldown + 迟滞（§3.3/§4） |
| 窄事件被窗口平滑 | `W` 过大时尖峰被均值稀释 | `W` 取「比目标尖峰宽 2~4 倍」；尖峰靠判据 B 的极值而非均值兜底 |
| 非等步长时间轴 | `W` 点数 ≠ 固定时长 | 用 `T/dt` 估 `W`，或对非等步长按时间界定窗口 |

## 9. 参数推荐（基于 tests/test.csv，50kHz）

| 参数 | 推荐值 | 说明 |
|---|---|---|
| `T` | 0.1–0.5 ms（5–25 点） | 比浪涌脉冲宽略大，兼顾平台 |
| `theta_avg` | 0.20（进） / 0.10（退，迟滞） | 均值突变阈 |
| `theta_ext` | 0.35 | 极值偏离阈 |
| `theta_end` | 0.50 | 事件结束相对回落阈；均值回落到活跃水平 (1-θ) 以下即结束（§3.3） |
| `sleep_floor` | 400 µA（= `0.4` mA 记法） | 唯一物理闸门 `floor=sleep_floor`；< 此值视为睡眠态，免疫所有判据（§3.4） |
| `cooldown` | `W` | 防抖 |
| `merge_gap` | 0.2 ms | 尖峰簇合并 |
