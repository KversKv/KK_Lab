

UI位置:
"脚本序列"本质是 Quick Commands 的进阶版（多条命令 + 时序 + 条件控制），所以最自然的做法是：与 Quick Commands 平级，作为底部面板的一个新 Tab。

二、推荐方案：底部面板新增 Tab（首选 ⭐）
在底部区域，把现有的 Quick Commands 改造为多 Tab 容器：

┌──────────────────────────────────────────────────────────┐
│ [ ⚡ Quick Commands ]  [ 📜 Scripts ]                       │  ← Tab 切换
├──────────────────────────────────────────────────────────┤
│  Script: [下拉选择脚本 ▼]  [▶ Run] [⏸] [⏹] [+ New] [Edit] │  ← 工具栏
│  Loop: [☐] x[1]   Status: ● Idle / Running (Step 2/5)      │
├──────────────────────────────────────────────────────────┤
│  Step │ 类型    │ 内容           │ 延时/条件   │ 状态       │
│   1   │ Send    │ AT             │ 100ms       │ ✓ Done     │
│   2   │ Wait    │ 等待 "OK"      │ 2000ms      │ ▶ Running  │
│   3   │ Delay   │ —              │ 500ms       │  Pending   │
│   4   │ Send    │ AT+RST         │ —           │  Pending   │
│   5   │ Loop    │ 回到 Step 1    │ x3          │  Pending   │
└──────────────────────────────────────────────────────────┘
优点：复用现有视觉风格，不增加学习成本，与快捷命令逻辑统一。



用户需求案例:
需要定时批量发生指令集的功能，可设置脚本，例如：
# wifi压测需求：
1. 扫描：wscan，输入此指令后等待8s；
2. 连接：wsconn BES-2F-Guest UVWxyz@@2023，输入此指令后等待20s；
3. 断连：wsdisconn，输入此指令后等待2s；
4. 卸载wifi：wdeinit ，输入此指令后等待4s；
5. 加载wifi：winit，输入此指令后等待5s；
6. reboot复位：reboot，输入此指令后的等待5s；

第一栏表示指令；第三栏表示优先级，开启后，按照从小到大的顺序执行，写0表示不执行；第四栏表示等待时间；
