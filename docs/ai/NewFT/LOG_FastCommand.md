# 任务：精简重构 PyQt6 串口工具的快捷指令区域

你是 Python + PyQt6 开发助手。请直接修改当前项目代码，只重构「快捷指令」区域，不要重写整个串口工具。

## 背景

当前是一个 PyQt6 串口工具，已有串口打开、关闭、发送、接收、日志显示等功能。

现在快捷指令全部混在一起，不同项目无法区分。请把快捷指令改成：

```text
项目 -> 区域 -> 分组 -> 指令
```

例如：

```text
项目A
└── 区域A
    └── 分组A
        ├── 查询版本
        └── 重启设备
```

## 重要要求

1. 使用 PyQt6，不要更换 GUI 框架。
2. 不要破坏现有串口功能。
3. 快捷指令点击发送时，必须复用现有串口发送逻辑。
4. 不需要兼容旧快捷指令数据。
5. 不需要旧数据迁移。
6. 不要做复杂功能，例如收藏、右键菜单、危险确认、拖拽排序、搜索、全局指令等。
7. 所有快捷指令配置只保存到一个 JSON 文件。

## 配置文件

使用一个配置文件：

```text
quick_commands.json
```

保留当前打开的检测逻辑;

JSON 结构如下：

```json
{
  "version": "1.0",
  "last_project_id": "project_default",
  "last_region_id": "region_default",
  "last_group_id": "group_default",
  "projects": [
    {
      "id": "project_default",
      "name": "默认项目",
      "regions": [
        {
          "id": "region_default",
          "name": "默认区域",
          "groups": [
            {
              "id": "group_default",
              "name": "默认分组",
              "commands": []
            }
          ]
        }
      ]
    }
  ]
}
```

每条指令格式：

```json
{
  "id": "cmd_xxx",
  "name": "查询版本",
  "content": "AT+GMR",
  "send_type": "text",
  "line_ending": "\\r\\n",
  "encoding": "utf-8"
}
```

字段只保留这些，不要增加复杂字段。

## 启动逻辑

程序启动时：

1. 读取 `quick_commands.json`。
2. 如果不存在，则创建默认配置。
3. 根据 `last_project_id`、`last_region_id`、`last_group_id` 恢复上次选择。
4. 只显示当前项目、当前区域、当前分组下的快捷指令。
5. 不要把所有项目的指令混在一起显示。

## UI 要求

快捷指令区域保持简单，包含：

```text
项目下拉框
区域下拉框
分组下拉框

[新增项目] [新增区域] [新增分组]
[新增指令] [导入] [导出]

快捷指令按钮区域
```

建议控件：

- `QComboBox`：项目、区域、分组选择
- `QPushButton`：新增、导入、导出、快捷指令按钮
- `QScrollArea + QGridLayout`：显示快捷指令按钮
- `QInputDialog`：新增项目、区域、分组
- `QDialog`：新增指令
- `QFileDialog`：导入导出
- `QMessageBox`：提示信息

  最顶层的分组应该是在Quick Commands右侧的标签栏样式和对应管理逻辑;

## 功能要求

### 1. 新增项目

点击「新增项目」：

1. 输入项目名称。
2. 创建新项目。
3. 新项目默认包含：

   - 默认区域
   - 默认分组
4. 保存配置。
5. 自动切换到新项目。

### 2. 新增区域

点击「新增区域」：

1. 输入区域名称。
2. 在当前项目下创建区域。
3. 新区域默认包含：

   - 默认分组
4. 保存配置。
5. 自动切换到新区域。

### 3. 新增分组

点击「新增分组」：

1. 输入分组名称。
2. 在当前项目、当前区域下创建分组。
3. 保存配置。
4. 自动切换到新分组。

### 4. 新增指令

点击「新增指令」后弹出窗口，只需要填写：

```text
指令名称
指令内容
发送方式：text / hex
结尾符：无 / \r / \n / \r\n
编码：utf-8 / gbk / ascii
```

保存后，把指令加入：

```text
当前项目 -> 当前区域 -> 当前分组
```

然后保存 `quick_commands.json` 并刷新按钮区域。

### 5. 切换项目、区域、分组

切换任意下拉框后：

1. 刷新下级下拉框。
2. 刷新快捷指令按钮。
3. 更新 `last_project_id`、`last_region_id`、`last_group_id`。
4. 保存配置。

### 6. 快捷指令发送

点击快捷指令按钮后：

1. 读取 `content`。
2. 根据 `send_type` 判断 text 或 hex。
3. 根据 `line_ending` 追加结尾符。
4. 根据 `encoding` 处理文本编码。
5. 调用当前已有的串口发送函数。
6. 不要重新写一套串口发送逻辑。

## 导入功能

点击「导入」选择 JSON 文件。

只支持新格式：

```json
{
  "version": "1.0",
  "projects": []
}
```

导入必须是增量导入：

1. 不覆盖已有项目。
2. 不覆盖已有区域。
3. 不覆盖已有分组。
4. 不覆盖已有指令。
5. 项目同名则合并。
6. 区域同名则合并。
7. 分组同名则合并。
8. 指令同名则自动重命名。
9. ID 冲突则自动生成新 ID。

同名指令重命名规则：

```text
查询版本
查询版本_导入
查询版本_导入_1
查询版本_导入_2
```

导入完成后提示：

```text
导入完成
新增项目：x
新增区域：x
新增分组：x
新增指令：x
重命名指令：x
```

## 导出功能

点击「导出」时，只导出当前项目。

导出格式：

```json
{
  "version": "1.0",
  "projects": [
    {
      "id": "当前项目ID",
      "name": "当前项目名称",
      "regions": []
    }
  ]
}
```

文件名建议：

```text
当前项目名称_快捷指令.json
```

## 保存要求

1. JSON 使用 UTF-8。
2. 保存时使用：

```python
json.dump(data, f, ensure_ascii=False, indent=2)
```

3. 配置文件不存在时自动创建。
4. 配置文件损坏时用 `QMessageBox` 提示，不要让程序崩溃。

完成后需要满足：

1. 快捷指令按项目、区域、分组区分。
2. 启动后恢复上次选择。
3. 只显示当前分组下的指令。
4. 可以新增项目、区域、分组、指令。
5. 点击指令可以正常通过现有串口发送逻辑发送。
6. 导入不会覆盖已有指令。
7. 导出只导出当前项目。
8. 所有快捷指令数据只保存在一个 `quick_commands.json` 文件中。
