# PMU 1811

BES1811 PMIC 图形化配置工具：通过 USB 转 I2C（设备地址 0x17，10 位寄存器地址，16 位数据）控制各 LDO / BUCK 的使能、模式与输出电压，含拓扑可视化与属性面板。

> 完整开发文档见 [ui/pages/pmu/pmu_1811/README.md](../../../ui/pages/pmu/pmu_1811/README.md) 与 [RULES.md](../../../ui/pages/pmu/pmu_1811/RULES.md)。本页只介绍最终用户使用。

## 页面入口

- 导航栏 → `TOOLS` → `PMU` → 子菜单 `1811`
- 对应源码：`ui/pages/pmu/pmu_1811/page.py`（`Pmu1811UI`）
- 独立预览：`python -m ui.pages.pmu.pmu_1811.page`

## 界面布局

### 1. 顶部标题栏 + Check 按钮
- 标题：`KK'1811 PMU 配置工具`。
- Check 按钮：触发读取全部 LDO/BUCK 当前状态（使能、模式、电压）。

### 2. 顶部 Chip Config 区
四个功能按钮（当前占位，仅记 WARN 日志）：
- Auto
- Force Normal
- Force RC
- Force Sleep

### 3. 中央 — 拓扑画布（DiagramCanvas）
- **VSYS 主母线**（琥珀色实线）：跨所有 VSYS 取电的模块。
- **一级模块卡片**（左列）：直接从 VSYS 取电的 LDO/BUCK。
- **二级模块卡片**（右列）：通过子母线级联（如 `vdd_l14_15` / `vdd_l5`）。
- **子母线**（蓝色半透明）：`vdd_l14_15` → LDO_14/15/VMIC1/VMIC2；`vdd_l5` → LDO_05/13。
- **并联对偶短接线**（紫色半透明）：`BUCK_01↔LDO_01` / `BUCK_02↔LDO_02` / `BUCK_03↔LDO_03` / `BUCK_06↔LDO_06` 四组输出短接（互斥使能）。
- **子母线药丸**：深灰底 + 蓝字，标注子母线名称。
- 每张卡片含：模块 ID、LED 状态灯、名称、电压步进 +/−、齿轮（属性面板入口）。

### 4. 右侧 — Property Panel（属性面板，默认隐藏）
- 选中卡片后显示该模块属性：
  - Enable 开关（iOS 风格拨动开关）
  - Mode 选择（Normal / LP；BUCK 还支持 ULP）
  - Voltage 滑块 + Spin 输入（对齐到 step）
  - I2C 信息（设备地址、寄存器映射、当前 pu_status 等）

### 5. 底部 — Execution Logs
- 打印每次 I2C 读写操作、连接状态、错误信息。

## 典型操作流程

### 流程一：首次连接并 Check
1. 进入本页 → UI 完全布局后自动触发 `QTimer.singleShot(0, self._on_check)`。
2. `LdoReadAllWorker` 启动 → `Bes1811PmuController.connect()` → `read_all_ldos()`。
3. Worker 完成 → 状态写回 `_modules` → 所有卡片刷新 + 属性面板（如已选模块）刷新。
4. 若 I2C 连接失败，日志区显示 `I2C 接口初始化失败`。

### 流程二：手动改电压
1. 单击卡片 → 右侧属性面板出现。
2. 在属性面板拖动 Voltage 滑块或输入数值 → 自动对齐到 step。
3. 释放滑块 / 回车 → 触发 `voltage` 写入 Worker。
4. Worker 通过 I2C 写入 `vbit_normal` 寄存器 → 完成后卡片电压显示更新。

### 流程三：切换使能（含对偶互锁）
1. 选中 `BUCK_01` 卡片 → 属性面板拨开 Enable 开关。
2. 因 `BUCK_01↔LDO_01` 是对偶，触发 `PairWriteWorker`：
   - 一次 I2C 会话内先开 `BUCK_01`、再关 `LDO_01`
3. 完成后两卡片状态同步刷新（BUCK_01 开、LDO_01 关）。
4. 关闭 `BUCK_01` 则走普通 `LdoWriteWorker`，不影响对偶。

### 流程四：右键菜单快速操作
- 右键卡片 → 弹出 `ContextMenu`：
  - 切换使能（与面板开关等效）
  - 选择模式（Normal / LP / ULP）

## 关键参数说明

| 参数 | 单位 | 说明 |
|---|---|---|
| Voltage | V | LDO/BUCK 输出电压，对齐到 step |
| Mode | — | Normal / LP（BUCK 还有 ULP） |
| Enable | bool | 模块使能状态 |
| Step | mV | 电压步进（来自芯片表 `get_ldo_step`） |

## 注意事项

- **并联对偶互锁**：`BUCK_01↔LDO_01` 等 4 组输出短接，开启一个会自动关闭对偶，**不可同时驱动**（硬件冲突）。
- **首次自动 Check**：页面首次显示会自动 Check 一次；如 I2C 未连接，会显示错误但不阻塞 UI。
- **写入保护**：`_start_write` 内依次检查：①模块是否可控制（`controllable`）；②I2C 是否已连接；③上次操作是否未完成。任一不满足则丢弃本次写入。
- **连接参数**：通过 `_dll_path` / `_speed_mode` 注入；默认 `None` 时使用 controller 默认 DLL 与 100K 速率。
- **Chip Config 按钮**：当前功能留空（仅记 WARN 日志），后续版本实现。
- **颜色含义**：
  - 琥珀 `#f59e0b` — VSYS 主轨
  - 蓝 `#3b82f6` — 子母线级联
  - 紫 `#a855f7` — 并联对偶输出短接
  - 翡翠绿 `#10B981` — 使能/选中/数值
- **Mock 模式**：`DEBUG_MOCK=True` 时 I2C 走 Mock，拓扑显示正常，但状态为模拟。
- **不要硬编码模块**：新增 LDO/BUCK 需在 `chips/bes1811_pmu.py` 加寄存器映射 + `models.py::_LAYOUT_ROWS` 加布局行，`_default_modules()` 会自动构建。
